# Security Policy

## Reporting a Vulnerability

If you discover a security issue, do not open a public GitHub issue with exploit details.

Report the issue privately to the project maintainers with:

- A clear summary of the problem
- Steps to reproduce
- Impact assessment
- Suggested remediation if available

## Scope

This project includes Firebase-backed authentication, user data, addresses, and order workflows. Vulnerabilities affecting access control, data exposure, or privileged admin operations should be treated as high priority.
