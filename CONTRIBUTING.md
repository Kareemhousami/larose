# Contributing

## Scope

Contributions should stay focused, production-oriented, and aligned with the app's existing architecture.

## Before Opening a Pull Request

1. Keep changes scoped to the problem you are solving.
2. Avoid committing generated files, local reports, or environment-specific artifacts.
3. Run:

```powershell
flutter analyze
```

4. Fill out the pull request template clearly.

## Code Standards

- Follow the existing Flutter, Provider, and Firebase patterns already used in the repository.
- Prefer small, reviewable pull requests over broad refactors.
- Keep public repository content clean and relevant to the product.
